Отличный вопрос! Вот open-source аналоги под каждый слой стека Apple:

---

## Фреймворк / UI

Apple использует собственный компонентный фреймворк с UMD-модулями и `.built.js` бандлами без тяжёлого рантайма. Ближайший аналог по философии — **Vanilla JS + Web Components** или **Lit** (Google). Если хочешь чуть больше структуры — **Svelte**, он компилируется в чистый JS без рантайма, что даёт такой же лёгкий результат как у Apple.

**Рекомендация:** `Svelte` или `Lit`

---

## Сборка

Файлы вида `main.built.js` без следов Webpack/Vite — это кастомный пайплайн. Из open-source ближе всего по скорости и минималистичности — **Vite** (использует Rollup под капотом). Или просто **esbuild** — молниеносный бандлер без лишнего.

**Рекомендация:** `Vite` + `Rollup`

---

## CSS / Дизайн-система

Apple активно использует CSS Custom Properties (`--sk-body-text-color`, `--sk-fill-red` и т.д.) — это их собственная дизайн-токен система. Open-source аналог — **Open Props** (готовые CSS-переменные) или просто собственные токены поверх **PostCSS**. Для сетки и адаптивности — чистый CSS Grid/Flexbox без Bootstrap.

**Рекомендация:** `PostCSS` + `Open Props` или кастомные CSS-переменные

---

## Шрифты и иконки

Apple использует проприетарные Apple Icons шрифты. Аналоги: **Font Awesome** (иконочный шрифт), **Phosphor Icons** или **Lucide** (SVG-иконки, более современный подход).

**Рекомендация:** `Lucide` или `Phosphor Icons` (SVG)

---

## Адаптивные изображения

Apple использует `<picture>` элементы — это стандарт HTML5, просто используй то же самое. Для автоматической оптимизации — **sharp** (Node.js) или плагин для Vite.

**Рекомендация:** `<picture>` + `sharp` для обработки

---

## Аналитика

Adobe Analytics → open-source замена: **Umami** (self-hosted, простой, GDPR-friendly) или **Plausible** (платный, но есть self-hosted версия). Если нужно что-то мощнее — **Matomo**.

**Рекомендация:** `Umami` (бесплатный self-hosted)

---

## Итоговый стек

```
Svelte или Lit          ← компоненты без тяжёлого рантайма
Vite + Rollup           ← сборка
PostCSS + CSS Variables ← стили и дизайн-токены
Lucide Icons            ← иконки (SVG)
<picture> + sharp       ← адаптивные изображения
Umami                   ← аналитика
HTTP/2 + Nginx          ← сервер (как у Apple — h2)
Akamai / Cloudflare CDN ← CDN (Cloudflare бесплатен)
```

По духу это максимально близко к тому, что делает Apple: минимум зависимостей, много нативного CSS и JS, быстрая загрузка, без тяжёлых фреймворков типа React/Next.js.
